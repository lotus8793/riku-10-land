# 申し送りノート 2026-06-09

## 概要

りくの10ランドに「しゅぎょう」タブ（答えが10未満の足し算モード）を新設し、きろくタブをモード別3列ランキングに更新した。

---

## 変更ファイル

`index.html` / `app.js` / `styles.css`

---

## 主な変更点

### 1. タブ構成を3→4に変更

| data-mode | 表示名 | 内容 |
|-----------|--------|------|
| `simple` | しゅぎょう | **新設** 答えが9以下の足し算（1+1〜8+1 など） |
| `pair` | あわせて10 | 旧「しゅぎょう」。10の補数ペア選択 |
| `bridge` | ぼうけん | 変更なし。さくらんぼ計算 |
| `records` | きろく | 3列ランキングに拡張 |

デフォルト起動モードは `simple`（旧: `pair`）。

### 2. しゅぎょうモード（simple）の仕様

- **問題**: `a + b` の形。a, b ∈ 1〜8、a+b ≤ 9（36問プール）
- **選択肢**: 1〜9 を常に3×3グリッドで表示
- **10マス表示**:
  - 問題中: 10マス全て空（白）
  - 正解後: a（緑）＋ b（赤）＋ 残り（白）でアニメーション表示
- **式の表示**:
  - 問題中: `3 + 4`（プレーンテキスト）
  - 正解後: `<span class="eq-green">3</span> + <span class="eq-red">4</span> = 7`（あわせて10と同形式）

### 3. きろくの分離

モードごとに localStorage キーを分けて管理：

```
riku10-records-simple   // しゅぎょう
riku10-records-pair     // あわせて10
riku10-records-bridge   // ぼうけん
```

旧キー `riku10-challenge-records` は廃止（マイグレーションなし）。

きろくタブは3列表示（日付は非表示、スコアのみ）。

### 4. 追加した関数（app.js）

| 関数名 | 役割 |
|--------|------|
| `makeSimpleProblems()` | 36問のプール生成 |
| `simpleProblemKey(p)` | 問題のキー文字列（連続出題防止用） |
| `makeSimpleChoices()` | `[1..9]` を返す |
| `nextSimple()` | 次の問題を描画 |
| `chooseSimple(value, button, problem)` | 解答処理 |

### 5. 既存関数の更新箇所

- `timerElements(mode)` — `"simple"` 対応追加
- `renderTimerRows()` — simple タイマー追加
- `setNextButton(mode, visible)` — simple 対応
- `modeSection(mode)` — simple 対応
- `handleChallengeEnd()` — simple タイマーロック追加
- `startMode(mode)` — `nextSimple()` 呼び出し追加
- `switchMode(mode)` — `#simple-mode` の表示切替追加
- `addRecord(score, mode)` — mode が `"simple"` のとき `simpleRecords` に保存
- `saveRecords()` / `renderRecords()` — simple 対応
- `reset-progress` ハンドラ — `simpleStreak` リセット追加

## 注意事項

- `app.js` の先頭付近で `simpleProblems = makeSimpleProblems()` を `makeBridgeProblems()` の直後に呼んでいる。`makeSimpleProblems` 関数の定義はその後に来るが、`var`/`const` の巻き上げの関係で **関数宣言（function 文）でないと動かない**。現在は function 宣言で書いてあるので問題なし。
- きろく列の日付（`.record-date`）は CSS で `display: none` にしている。3列では幅が足りないため。必要なら列数を減らすか表示を変える。
